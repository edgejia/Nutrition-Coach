# Codex / Claude Runtime Parity Contract

The machine-readable source is [`runtime-parity.json`](runtime-parity.json). `yarn workflow:runtime-parity` rechecks the live local GSD installs, pinned per-host exact skill name/tree-digest manifests, pinned project-instruction bytes plus read-only queries, host registry fields, frozen shared-config digest, the closed SHA-256 manifest for the project verifier entrypoint and all of its local transitive modules (including `state-check.mjs`), and a runtime-neutral good/bad proof fixture. Its final freshness snapshot repeats every core, every file and directory identity in each governed skill tree, instruction, config, matrix, project-verifier, registry-bearing core, and smoke-fixture input. File evidence requires a single link and binds digest plus `dev`/`ino`/link-count/mode/size/mtime/ctime identity, so an off-tree alias or A-to-B-to-A substitution does not become fresh merely because bytes and mtime were restored. Any live-check failure forces `readiness: not_ready` even when the matrix has no blocking row. The result exposes `matrixSha256`, `projectVerifierBundleSha256`, and `evidenceSnapshotSha256`; these values are review evidence, not an independent trust root, so the approved maintenance commit and reviewed matrix bytes must remain the source-release boundary.

The matrix also pins the complete expected output of the project wiring checker. At this maintenance boundary the only accepted findings are the two explicit missing `nutrition-planning-proof` role bindings, one for `gsd-planner` and one for `gsd-plan-checker`. Every additional finding, missing expected finding, or changed finding payload makes the parity check fail with a stale-matrix finding; a generic blocking status cannot mask a new cause. Before the maintenance files are committed, the live check is expected to fail because the guidance and skill are not yet tracked source. From the reviewed maintenance commit, the intended deterministic result is `status: pass` with `readiness: not_ready`: the exact two expected wiring findings remain documented blockers until an independently approved activation, and the blocking/deferred matrix rows still prevent readiness.

Matrix statuses have narrow meanings:

- `equivalent`: the stated bounded claim has matching evidence.
- `intentional_difference`: host behavior differs by design and carries an explicit rationale.
- `blocking`: local tooling may exist, but the project cannot rely on the behavior yet.
- `deferred`: evidence requires an approved pilot or later representative workflow samples.

Equal core hashes prove only installed-core identity. They do not make shared `.planning/config.json` an authoritative host signal, prove skill injection, or erase Codex/Claude dispatch differences. Artifact provenance comes from the active lease token; the shared `runtime` key is a configuration projection only.

The current matrix deliberately leaves artifact-writer wiring, lease entrypoint coverage, model/effort propagation, and representative telemetry unresolved. [`runtime-governance.md`](runtime-governance.md) defines the implemented lease/provenance/telemetry controls and [`gsd-reenable-pilot.md`](gsd-reenable-pilot.md) bounds the still-unapproved behavioral pilot. No matrix result authorizes `.planning` mutation in this checkout, GSD resume, GitHub writes, production, migration, runtime refresh, Tunnel, smoke, merge, or tag actions.
