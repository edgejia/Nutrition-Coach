## Trust Boundaries

| Boundary | Description |
|---|---|
| `/api/history/trends` -> History aggregate UI | Trend aggregates are useful display facts but not snapshot authority. |
| `/api/history/days/:date` -> timeline rows/edit identity | Snapshot meals are the only source for rows, Meal Edit payloads, confirmed empty state, and Day Detail activation. |
| Zustand `lastMealMutation` -> History caches | Affected-date mutation notices may trigger refresh/invalidation and must stay scoped. |
| Pending UI -> user action | Pending visuals must not offer stale or trends-only row/detail/edit actions. |
