<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM tool call -> `runContract()` | Untrusted model JSON crosses into validated backend tool contracts. |
| `runContract()` -> `executeTool()` | Contract-specific results must be adapted before the orchestrator sees them. |
| `executeTool()` -> `server/orchestrator/index.ts` | Only typed `ToolExecutionResult` facts and renderer `controlledReply` may cross; raw `contractResult` is blocked by D-01. |
| renderer copy -> route persistence | Terminal clarification text must be safe to persist for follow-up context without raw prompt/tool/provider evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-01 | Tampering | LLM tool clarification status | mitigate | Red-first tests require `executeTool()` to expose validated typed clarification facts, not parsed model-authored tool-message JSON. |
| T-68-02 | Spoofing | Historical clarification final reply | mitigate | Orchestrator tests require `finalReplySource === "renderer"` and one LLM call for terminal historical clarification. |
| T-68-03 | Information Disclosure | Source guard / proof assertions | mitigate | Tests assert no raw `contractResult`, serialized tool payload, prompt, user text, or assistant final text leaks into `index.ts` behavior. |
| T-68-04 | Tampering | `multiple_targets` carry-forward | mitigate | Tests require multi-date summary copy not to seed `extractPreviousHistoricalDateKey()` as a single resolved prior date. |
| T-68-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned; RESEARCH.md states package legitimacy gate is not applicable. |
</threat_model>
