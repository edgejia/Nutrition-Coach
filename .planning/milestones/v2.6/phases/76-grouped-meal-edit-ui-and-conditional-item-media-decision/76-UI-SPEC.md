> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none - use existing custom Sport UI primitives |
| Preset | not applicable |
| Component library | Custom `SportScreen`, `SportCard`, `SportIconButton`; no Radix/base-ui |
| Icon library | Existing `SportIcons` only; use `SportChevronLeftIcon`, `SportPlusIcon`, and `SportCloseIcon` where icon affordances are needed |
| Font | `var(--sp-font-zh)` for user-facing copy, `var(--sp-font-mono)` for compact labels and numeric summaries |

Source: `client/src/components/SportPrimitives.tsx`, `client/src/components/SportIcons.tsx`, `client/src/app.css`, `76-CONTEXT.md`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, dense row metadata separators |
| sm | 8px | Compact row gaps, item-list gaps, field helper text spacing |
| md | 16px | Default card gap and grouped editor section rhythm |
| lg | 24px | Major vertical separation within the edit scroll |
| xl | 32px | Reserved for larger screen breathing room only |
| 2xl | 48px | Header grid side columns and stable action zones |
| 3xl | 64px | Reserved; do not add new page-level gaps in Phase 76 |

Exceptions: Preserve existing Meal Edit inherited values only: `18px` horizontal screen padding, `10px` field gaps/radius, `38px` current back button size, and `46px` footer button height. New grouped row controls must use 44px minimum tap targets and must not introduce additional non-scale spacing.

## Typography

Use exactly these four sizes and two weights for new grouped editor UI: 400 and 700.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 10px | 700 | 1.25 |
| Supporting Body | 12px | 400 | 1.5 |
| Body / Input | 14px | 400 | 1.45 |
| Heading | 20px | 700 | 1.2 |

No viewport-scaled font sizing. Letter spacing is allowed only where existing `sp-meal-edit-*` mono labels already use it; new Chinese row copy uses `letter-spacing: 0`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--sp-bg)` / `#0a0b0d` | Full-screen Meal Edit background and footer gradient base |
| Secondary (30%) | `var(--sp-surface)` / `#131418`; `var(--sp-surface-2)` / `#1c1e23` | Cards, collapsed item rows, expanded field containers, image frame |
| Accent (10%) | `var(--sp-lime)` / `#d6ff3a` | Save CTA, Add item CTA, expanded row border, focus outline, grouped section label |
| Destructive | `var(--sp-red)` / `#ff4d4d` and `#ffb3b3` text | Row delete, whole-meal delete, blocked final-row delete message, mutation error alert |

Accent reserved for: `儲存餐點`, `新增項目`, the single expanded row border, keyboard focus outlines, and the grouped editor label. Do not use accent for every editable row or every numeric field.

## Component Inventory

| Surface | Contract |
|---------|----------|
| Screen shell | Reuse the current absolute full-screen Meal Edit overlay, `SportScreen`, existing centered header title `編輯餐點`, origin-specific back labels, scroll-safe main area, and fixed footer. |
| Whole-meal media | Keep `MealEditImageFrame` above the editor. Copy remains whole-meal: `整餐照片` and `這張照片代表整餐，不是單一食物裁切。` No row may show image thumbnails, crop handles, media badges, or item-evidence copy. |
| Grouped editor card | Replace `sp-meal-edit-grouped-lock` content with a grouped edit card using the same surface, border, and compact rhythm. Do not keep `到對話修正` as the primary grouped path when authoritative `items[]` are present. |
| Live totals strip | Show draft aggregate totals near the grouped card header: calories, protein, carbs, fat. Values update from draft `items[]` and must not display original totals as current after edits. |
| Collapsed item row | One row per item. Show `name · calories kcal · Pprotein · Ccarbs · Ffat` on one compact wrapping line. Row has an edit/expand affordance and a delete affordance; collapsed rows do not contain inputs. |
| Expanded item row | Only one row expanded on mobile. Expanded row contains exactly `name`, `calories`, `protein`, `carbs`, `fat`; `position` is not visible or editable. Use the existing name field and two-column macro grid pattern. |
| Add item | Place below the item list inside the grouped editor card. It appends an empty row, expands it immediately, and uses accent treatment plus `SportPlusIcon` if an icon is used. |
| Footer actions | Preserve fixed footer: cancel on the left, primary save on the right. Save submits the full ordered draft only after validation passes. |

Primary focal point is the grouped editor card header with live aggregate totals; the expanded row accent border is the secondary focus.

## Interaction Contract

1. Opening a grouped meal with authoritative `items[]` shows the whole-meal image frame, live aggregate totals, collapsed item rows, Add item, whole-meal Delete, Cancel, and Save.
2. Opening a grouped meal without item details shows the empty/unsupported state copy above and does not render editable fields or imply that a mutation can succeed.
3. Tapping a collapsed row expands that row and collapses any previously expanded row.
4. Editing values updates only local draft state until Save.
5. Save validates all rows. If any row is invalid, block submit, open the first invalid row, show field-level errors, and show the top-level invalid save error.
6. Successful Save calls grouped `updateMeal` with complete ordered `items[]`, then reuses `refreshAfterMealMutation`, closes Meal Edit, and relies on refreshed authoritative DTOs.
7. Non-final row delete removes the draft row immediately without confirmation. It does not call the API until Save.
8. Final row delete is blocked inline with the final item delete block copy and does not call `PATCH` or `DELETE`.
9. Whole-meal Delete remains the existing destructive action with confirmation and `DELETE /api/meals/:id`.
10. Back/Cancel exits immediately only when the grouped draft is unchanged. Dirty edits, adds, or deletes prompt once with the dirty discard confirmation.
11. Stale revision conflicts reuse the existing stale-blocked state: disable Save and Delete, show top-level stale copy, refresh affected today rows when applicable, and offer `重新載入餐點`.

## Validation States

| State | Visual Contract |
|-------|-----------------|
| Valid collapsed row | Surface-2 row, no accent border, compact summary text in ink and ink-2. |
| Expanded row | Accent border using `var(--sp-lime-line)` or `var(--sp-lime)`, inputs visible, delete still row-scoped. |
| Field error | Keep row expanded; field container gets destructive border/background, helper copy appears below the field in `#ffb3b3`. |
| Top-level save error | Reuse `sp-meal-edit-error` alert styling and `role="alert"`. |
| Pending save/delete | Disable row controls, Add item, Save, Cancel where existing behavior disables exit during mutation; button labels use existing pending style such as `儲存餐點中...`. |
| Stale blocked | Disable Save and Delete. Keep the current stale alert and `重新載入餐點` recovery button. |

## Accessibility Contract

- All row expand controls and row delete controls must be native `button` elements.
- Icon-only controls require `aria-label`; acceptable labels are `展開項目：{name}`, `收合項目：{name}`, and `刪除項目：{name}`.
- Row delete and add controls must have at least 44px tap targets.
- The expanded row should expose the item name in visible text and not rely on color alone to indicate expanded state.
- Error text must be programmatically associated with the relevant input through `aria-describedby` when practical; the top-level error uses `role="alert"`.
- Keyboard order: back button, expanded/collapsed row controls in visible order, Add item, whole-meal Delete, Cancel, Save.

## Media Decision

Phase 76 explicitly defers item-level media mapping. The UI must preserve whole-meal photo identity and must not add:

- Per-item thumbnails.
- Per-item photo crop controls.
- Copy that says an item row is proven by a specific image region.
- New `MealItemDetail` media fields.
- Any grouped PATCH image field when saving item edits.

Source-of-truth note for downstream docs/tests: item-level photo mapping is deferred until a later phase defines persistence, DTO, and evidence contracts.
