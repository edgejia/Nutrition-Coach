## Trust Boundaries

| Boundary | Description |
|---|---|
| Built client -> CDP script | The visual proof drives the real built UI but must inject deterministic local mocks before app code runs. |
| Mocked API responses -> History UI | Synthetic History responses must exercise pending behavior without real user data. |
| Browser screenshot/manifest -> repo artifacts | Generated proof can leak sensitive content if it captures real state or over-detailed payloads. |
| Local proof -> release process | Evidence artifacts must not imply staging or production authorization. |

|---|---|---|---|---|

| T-77-02 | Elevation of Privilege | Trends-only detail/edit activation | mitigate | Script asserts pending state has inline loading copy and no snapshot-only row/detail affordance before delayed day response resolves. |
