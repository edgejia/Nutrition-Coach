<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `client/src/sse.ts` -> coordinator | Validated but still untrusted event metadata crosses into state orchestration. |
| coordinator -> REST API | Same-day SSE event triggers `/api/meals` row refetch. |
| coordinator -> Zustand actions | Coordinator commits rows, summaries, or historical invalidation into app state. |
| initial row load -> coordinator | Mount-time `getMeals()` can resolve out of order relative to SSE reconcile. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-04-01 | Tampering | same-day coordinator | mitigate | Require latest-token row refetch success before `setMeals` and `setDailySummary`. |
| T-63-04-02 | Tampering | initial row load | mitigate | Route initial `getMeals()` commits through the same token family so stale results are dropped. |
| T-63-04-03 | Tampering | historical event routing | mitigate | For `affectedDate !== todayKey`, never call `setDailySummary`, `setMeals`, or today's row refresh; use `recordMealMutation` for non-future historical dates. |
| T-63-04-04 | Denial of Service | future or repeated events | mitigate | Future events no-op; overlapping events are latest-wins without explicit coalescing. |
| T-63-04-05 | Information Disclosure | coordinator tests | mitigate | Tests use synthetic meal and summary objects only; no raw prompts, provider payloads, images, sessions, or DB snapshots. |
</threat_model>
