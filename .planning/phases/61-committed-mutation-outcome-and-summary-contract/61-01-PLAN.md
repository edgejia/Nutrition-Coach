| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Information Disclosure | `server/services/summary-outcome.ts` | mitigate | Keep recovery/unavailable handling metadata-only; tests must not persist raw user text, tool payloads, provider bodies, image data, sessions, or DB snapshots. |
| T-61-02 | Tampering | `server/services/meal-correction.ts` | mitigate | Preserve existing `deviceId`-scoped `updateTransaction` and `softDeleteTransaction` service calls; do not add raw meal id access paths. |
| T-61-03 | Repudiation | update/delete service results | mitigate | Return committed facts plus `summaryOutcome` after SQLite commit so post-commit freshness failure is distinguishable from mutation failure. |
| T-61-04 | Denial of Service | summary recompute/recovery | accept | Recovery attempts one persisted-meal read for the affected date; no retry loop or external service call is added. |
</threat_model>
