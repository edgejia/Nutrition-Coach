<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM tool-call JSON -> tool contract | Tool arguments are untrusted and validated before execution. |
| tool contract -> meal correction service | Numeric authority must pass before `mealCorrectionService.updateMeal()` writes a revision. |
| tool summaries -> chat history/trace | Summaries must stay redacted and metadata-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-66-09 | Tampering | `update_meal` top-level numeric args | mitigate | Run `authorizeMealNumericUpdate` before service writes and return controlled no-update copy on failure. |
| T-66-10 | Tampering | `update_meal.items[]` replacement | mitigate | Diff nested numeric values and require authorization for every changed nutrition field. |
| T-66-11 | Tampering | `propose_meal_numeric_correction` | mitigate | Accept only operator intent and resolved meal identity; compute numeric proposal values from current persisted meal facts. |
| T-66-12 | Denial of Service | stale/missing meal target | mitigate | Preserve existing resolved-target and Phase 62 revision precondition failure paths. |
| T-66-13 | Information Disclosure | tool log summaries | mitigate | Keep log summaries metadata-only; do not log raw user text, assistant final text, raw tool payloads, provider bodies, session material, image data, or DB snapshots. |
| T-66-SC | Tampering | npm/pip/cargo installs | accept | No package installation is planned; Package Legitimacy Audit lists none. |
</threat_model>
