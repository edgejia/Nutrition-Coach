<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user text -> backend parser | User-provided chat text is untrusted evidence and must be parsed deterministically. |
| LLM tool args -> meal write | Model-supplied `update_meal` args are untrusted and cannot become persisted nutrition facts without backend authority. |
| helper proof -> trace output | Tests must not persist raw provider bodies, session material, image data, or database snapshots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-66-01 | Tampering | `authorizeMealNumericUpdate` | mitigate | Require current-turn explicit numeric evidence for each changed meal numeric value before tool enforcement can write. |
| T-66-02 | Tampering | `items[]` replacement authorization | mitigate | Diff nested item numeric fields against current items and require the same authority as top-level fields. |
| T-66-03 | Spoofing/Tampering | prior assistant number confirmation | mitigate | For meal direct writes, reject previous assistant prose as authority unless later plans route it through stored backend proposals. |
| T-66-04 | Information Disclosure | unit proof output | mitigate | Use exact assertions and command output only; do not generate raw trace artifacts or database snapshots. |
| T-66-SC | Tampering | npm/pip/cargo installs | accept | No package installation is planned; Package Legitimacy Audit lists none. |
</threat_model>
