<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| User form input -> grouped draft helper | Item names and nutrition strings are untrusted until parsed and validated. |
| Browser grouped draft -> `PATCH /api/meals/:id` | Client must submit strict ordered `items[]`; server remains authoritative. |
| Server 409 conflict -> Meal Edit state | Stale revisions must block UI controls rather than closing as success. |
| Persisted item names -> JSX rendering | Names may contain arbitrary text and must render as text/input values only. |
| Whole-meal photo -> item rows | UI must not represent item-level media evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-76-07 | Tampering | `buildGroupedMealUpdateItems()` | mitigate | Helper derives zero-based positions from visible order and emits only strict `MealItemDetail` fields, preventing grouped over-posting from UI construction. |
| T-76-08 | Tampering | grouped stale save | mitigate | `MealRevisionConflictError` catch path reuses stale-blocked recovery and disables Save/Delete for grouped saves. |
| T-76-09 | Elevation of Privilege | unauthorized meal write | mitigate | `UNAUTHORIZED` continues to call `recoverGuestSession()` and no raw `deviceId` header/query selector is introduced. |
| T-76-10 | Information Integrity | item media evidence | mitigate | `MealItemDetail` source note, whole-meal image copy, and items-only grouped body prevent item-photo mapping claims in Phase 76. |
| T-76-11 | Information Disclosure | item-name XSS | mitigate | Render item names through JSX text and controlled input `value`; do not add `dangerouslySetInnerHTML`. Source tests from Plan 01 must pass. |
| T-76-12 | Denial of Service | invalid grouped drafts | mitigate | Client validation blocks blank/non-numeric/negative rows before network calls while server validation remains the final safety net. |
| T-76-SC | Tampering | package-manager installs | mitigate | No package installs are planned. If execution attempts npm/pip/cargo install, stop for package legitimacy audit first. |
</threat_model>
