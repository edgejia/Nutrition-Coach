<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| orchestrator result -> JSON route response | Backend terminal clarification becomes persisted assistant text and JSON payload. |
| orchestrator result -> SSE stream | Backend terminal clarification becomes streamed `chunk`/`done` payloads and persisted assistant text. |
| route mutation flags -> realtime publisher | `didMutateMeal` and summary facts control whether `daily_summary` can publish. |
| persisted assistant history -> future tool date parsing | Previous clarification copy can influence follow-up historical date carry-forward. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-09 | Tampering | JSON/SSE terminal clarification payloads | mitigate | Integration tests assert no mutation flags, no logged meal, no summary outcome, and no success-style copy. |
| T-68-10 | Repudiation | assistant reply persistence | mitigate | JSON/SSE tests query `/api/chat/history` after terminal clarification to prove `finalizeAssistantReply()` persisted the renderer copy. |
| T-68-11 | Tampering | `daily_summary` publish boundary | mitigate | Route tests spy on publisher and assert no publish for terminal clarification paths. |
| T-68-12 | Elevation of Privilege | historical date carry-forward | mitigate | Follow-up test proves multi-date summary clarification cannot grant authority to mutate/log against one historical date. |
| T-68-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned. |
</threat_model>
