<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service result -> renderer copy | Candidate facts become user-visible clarification text. |
| tool result -> orchestrator loop | Tool outcomes may terminate the model loop through controlled replies. |
| model tool call -> resolved target state | Non-resolved tool calls must not authorize later mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-01 | Elevation of privilege | `toolSessionState.resolvedMealTargets` | mitigate | Populate resolver-owned targets only for resolved `find_meals`; clear targets for clarification/not-found paths. |
| T-67-02 | Tampering | model retargeting after clarification | mitigate | Controlled replies terminate ambiguous target turns before the model can choose or rewrite. |
| T-67-03 | Tampering | stale revision overwrite | mitigate | This plan preserves mutator requirement for resolver-owned mealRevisionId and does not alter expected revision checks. |
| T-67-04 | Information disclosure | clarification renderers | mitigate | Renderer helpers omit raw correction text, inferred period labels, calories/macros, tool names, revision ids, and summary internals. |
| T-67-05 | Tampering | query safety | accept | This plan does not alter database queries; Plan 02 owns Drizzle-scoped candidate loading. |
| T-67-SC | Tampering | package installs | mitigate | No dependency additions. |
</threat_model>
