<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM tool call to `ToolContract` | Untrusted model arguments cross into mutation-capable backend code. |
| User text to consent/cancel predicate | User-visible consent or cancel terms determine whether pending state can be applied or cleared. |
| Mutation contract to realtime publisher | `goals_update` fan-out must happen only after persisted target mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-04 | Elevation of Privilege | `update_goals` proposal mode | mitigate | Require `mode`, backend consent predicate, active proposal state, and current-turn-only source checking for overrides before `deviceService.updateGoals`. |
| T-60-05 | Tampering | `propose_goals` contract | mitigate | Persist proposals as pending state only; never call `deviceService.updateGoals` or `publishGoalsUpdate` from proposal creation. |
| T-60-06 | Information Disclosure | tool log summaries | mitigate | Keep summaries to tool names, mode, field names, statuses, and event names; unit tests deny numeric values and raw text. |
| T-60-07 | Denial of Service | malformed tool args | mitigate | Zod validation rejects unknown/missing mode and out-of-range values before service calls. |
</threat_model>
