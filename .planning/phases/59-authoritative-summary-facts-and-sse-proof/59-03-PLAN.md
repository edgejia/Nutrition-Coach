<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Orchestrator result -> HTTP JSON response | Route may persist and send untrusted model text unless it composes summary/history facts first. |
| Orchestrator stream tokens -> SSE chunk frames | Incremental model tokens may leak before final guards unless held and composed. |
| Route session -> persisted fact output | Route must preserve existing signed-cookie device ownership and not introduce raw device IDs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-08 | Tampering | `server/routes/chat.ts` JSON/drained paths | mitigate | Compose from `summaryHistoryFacts` before `finalizeAssistantReply()` and `reply.send()`. |
| T-59-09 | Tampering | `handleStreamingReply()` SSE chunks | mitigate | Hold summary-context model tokens and emit only sanitized composer output before `done`. |
| T-59-10 | Information Disclosure | Route ownership scope | mitigate | Use existing cookie-backed `deviceId` resolution and injected services only; do not add query/header ownership paths. |
| T-59-11 | Repudiation | Saved assistant history | mitigate | Integration tests assert saved assistant history matches the route-composed response/chunk text. |
</threat_model>
