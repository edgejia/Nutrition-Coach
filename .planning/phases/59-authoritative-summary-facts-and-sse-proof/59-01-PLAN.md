<interfaces>
New module contract to create:
- `SummaryHistoryFacts`: `{ dailySummary?: DailySummary; meals: Array<{ foodName: string; calories: number }> }`.
- `renderSummaryHistoryFacts(facts, options?)`: returns deterministic fact text only. Per D-03, when `facts.meals` is non-empty, the visible meal count is `facts.meals.length` and the visible day kcal total is the sum of persisted meal kcal. `dailySummary` may supply date context and empty-day semantics, but mismatched aggregate `mealCount` or `totalCalories` must not override persisted meal-derived facts.
- `guardSummaryHistoryAdvice(advice, facts)`: returns accepted advice text or an empty string.
- `composeSummaryHistoryReply(facts, advice, options?)`: returns deterministic fact segment plus accepted advice.

Required exact deterministic current-day example:
- `今天已記錄 2 餐，共 900 kcal：豆腐飯 520 kcal、鮭魚飯 380 kcal。`

Empty-day semantics:
- A summary with `mealCount: 0`, `totalCalories: 0`, and no meals renders summary/history copy containing `0 餐` and `0 kcal`.
- It must not render mutation-failure copy such as `我還沒有把這餐寫入紀錄`.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM advice -> backend composer | Untrusted model advice crosses into final user-visible summary/history text. |
| Persisted meal facts -> renderer | Same-device stored meal rows become authoritative display facts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-01 | Tampering | `composeSummaryHistoryReply` | mitigate | Ignore model-authored meal facts and render deterministic meal names/kcal from `SummaryHistoryFacts.meals`. |
| T-59-02 | Tampering | `renderSummaryHistoryFacts` | mitigate | Keep aggregate day totals separate from per-meal facts; never use `dailySummary.totalCalories` as a per-meal value. |
| T-59-02B | Tampering | `renderSummaryHistoryFacts` aggregate reconciliation | mitigate | When aggregate `mealCount` or `totalCalories` disagrees with persisted meals, render the visible fact segment from persisted row count and persisted kcal sum. |
| T-59-03 | Information Disclosure | `guardSummaryHistoryAdvice` | mitigate | Drop advice containing concrete meal names, kcal, macros, meal count, or day totals so model text cannot leak or invent unsupported fact claims. |
| T-59-04 | Repudiation | Renderer tests | mitigate | Assert exact deterministic output for the canonical 2-meal case and explicit empty-day behavior. |
</threat_model>
