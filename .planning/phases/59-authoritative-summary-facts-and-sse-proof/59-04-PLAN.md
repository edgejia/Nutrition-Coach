Make the SSE ordering proof machine-checkable through stream close with structured-only artifacts.

Purpose: Promotion evidence must not stop at the first `event: done`; it must observe close and fail on any post-done `chunk` or `status` frame without storing raw SSE transcripts.

Output: Shared terminal proof helpers, unit negative tests, updated image-log-failure scenario proof, and regenerated structured artifacts.

Scope note: this plan intentionally lists the generated `tests/harness/artifacts/image-log-failure/latest/*.json` files with the harness helper/scenario changes because the proof is only complete when the scenario is rerun and the structured evidence is regenerated together. The generated files are not hand-edited; they are owned by Task 3's `yarn verify:harness -- image-log-failure` command and inspected for raw transcript leakage.
</objective>

<interfaces>
Existing SSE collection contract:
- `readStreamThroughClose(reader, options)` returns `{ raw, events, closed, firstDoneIndex, eventsAfterFirstDone, nonEmptyChunkBeforeDone, reads }`.
- `parseSSEEvents(raw)` returns ordered `{ event, data }` pairs.

New helper contract:
- `summarizeSSETerminalProof(collection)` returns structured metadata: `closed`, `firstDoneObserved`, `firstDoneIndex`, `noPostDoneChunkOrStatus`, `postDoneEventNames`, `terminalViolationEvents`, `nonEmptyChunkBeforeDone`, `readCount`, and `rawLength`.
- `assertSSETerminalProof(collection)` returns `{ ok: true, evidence }` or `{ ok: false, error, evidence }`; it fails when close is not observed, done is absent, or `chunk`/`status` appears after the first `done`.
</interfaces>
</context>
